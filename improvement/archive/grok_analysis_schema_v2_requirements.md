### スコアリングロジック

```
1. 基礎スコア = calculate_rank_score_improved()
   - 相対位置（上位25%: 40, 50%: 20, 75%: 0, 下位: -10）
   - バックテスト勝率補正（70%以上: +30, 30%以下: -20）

2. ファンダメンタルズ補正
   - ROE > 15%: +20
   - ROE < 0%: -15
   - 営業利益成長 > 50%: +25
   - 営業利益成長 < -30%: -20

3. テクニカル補正
   - 前日-3%以上下落: +15（リバウンド期待）
   - 前日+10%以上上昇: -10（過熱感）
   - ATR < 3.0%: +10（低ボラ）
   - ATR > 8.0%: -15（高ボラ）
   - 25日線から+5%以上: -10（割高）
   - 25日線から-5%以上: +10（割安）

4. 判断
   - スコア >= 40: 買い（高）
   - スコア >= 20: 買い（中）
   - スコア <= -30: 売り（高）
   - スコア <= -15: 売り（中）
   - それ以外: 静観
```

## 問題点

### 売りシグナル（81.8%勝率）✅ 優秀に機能している

- 22 件中 18 勝 4 敗
- 平均リターン+2.52%（ショート時）
- ROE < 0%（赤字）
- 営業利益成長 < -30%（減益）
- → **業績悪化 = 株価下落** が高精度で予測できている
- **→ v2.0 の売りシグナルは非常に優秀、触らない**

### 買いシグナル（44.6%勝率）❌ 改善が必要

- 65 件中 29 勝 36 敗
- 平均リターン-0.33%
- ROE > 15%（優良）
- 営業利益成長 > 50%（増益）
- → **業績好調 ≠ 短期株価上昇** とは限らない
- **Grok ランク勝率を軽視している**（勝率 60%超の銘柄を見落とす）
- **株価帯の影響を考慮していない**（5000-8000 円は勝率 62.5%なのに加点なし）

### 具体例（2025-11-19）

| 銘柄                     | v2 スコア  | v2 判断    | Grok 勝率 | 実際の優先度 |
| ------------------------ | ---------- | ---------- | --------- | ------------ |
| バリオセキュア           | 85（最高） | 買い（高） | 36.4%     | ❌ 低い      |
| シェアリングテクノロジー | 60         | 買い（中） | 63.6%     | ✅ 最高      |
| クラウドワークス         | 20（最低） | 買い（中） | 60.0%     | ✅ 高い      |
| トレックス               | 60         | 買い（中） | 9.1%      | ❌ 最低      |

**→ v2 スコアよりも Grok ランク勝率の方が信頼できる**

## 改善版 v2.0.1 価格帯スコア補正（最小限の改善）

**リリース日**: 2025-11-20
**データ期間**: 2025-11-04 〜 2025-11-18（15 日間、122 件）
**変更内容**: カラム追加なし、スコアリングロジックのみ修正
**目標**: 買い勝率 44.6% → 50-55%、売り勝率 81.8% 維持

### 背景

v2.0 の問題点分析の結果、**株価帯によって勝率が大きく異なる**ことが判明：

| 株価帯           | 件数  | 勝率      | 平均リターン | 判定          |
| ---------------- | ----- | --------- | ------------ | ------------- |
| **5000-8000 円** | **8** | **62.5%** | **+0.63%**   | ✅ 唯一プラス |
| 0-2000 円        | 77    | 36-41%    | -0.94%       | 平均的        |
| 2000-3000 円     | 15    | 26.7%     | -0.91%       | 低い          |
| 3000-5000 円     | 15    | 33.3%     | -0.76%       | やや低い      |
| 8000 円以上      | 7     | 14.3%     | -3.05%       | 非常に低い    |

**重要な発見**:

- 5000-8000 円の価格帯は勝率 62.5%、リターン+0.63%で唯一プラス
- 勝ち負け比率 1.83 倍（勝ち+3.05% vs 負け-1.67%）
- この価格帯に限定すれば、買い勝率目標 50%超を達成可能

### v2.0.1 スコアリング変更

**追加ロジック（1 つのみ）**:

```python
# 価格帯スコア補正（v2.0.1で追加）
if 5000 <= current_price < 8000:
    score += 25
    reasons.append("高価格帯（5000-8000円）: 買い勝率62.5%")
# その他の価格帯は補正なし（0点）
```

### なぜマイナス評価を加えないのか

**重要**: 低勝率価格帯（2000-3000 円など）にマイナス評価を加えると、売りシグナルが増える。

**v2.0 の売りシグナルは既に 81.8%勝率で非常に優秀**であり、これを壊すリスクは取れない。

売りシグナルの価格帯別分析：

| 株価帯       | 売り件数 | ショート勝率 | 判定       |
| ------------ | -------- | ------------ | ---------- |
| 0-1000 円    | 13       | 76.9%        | ✅ 優秀    |
| 1000-2000 円 | 3        | 100.0%       | ✅ 完璧    |
| 2000-3000 円 | 6        | 83.3%        | ✅ 優秀    |
| 3000 円以上  | 0        | N/A          | データなし |

**全ての価格帯で売りシグナルは高勝率**であり、現在のロジックが正しく機能している。

**→ マイナス評価を加えると、売りシグナルの精度を下げるリスクがある**

### v2.0.1 の戦略

1. **売りシグナルには一切手を付けない**（81.8%勝率を維持）
2. **買いシグナルの精度向上のみ**（5000-8000 円に加点）
3. **その他は静観判断に委ねる**（既存ロジック温存）

### 期待効果

| 指標                 | v2.0   | v2.0.1 目標       |
| -------------------- | ------ | ----------------- |
| 売り勝率（ショート） | 81.8%  | **81.8%維持**     |
| 売りリターン         | +2.52% | **+2.52%維持**    |

### 実装の容易性

- **カラム追加不要**: 既存の`current_price`のみ使用
- **コード変更最小**: スコアリング関数に 3 行追加
- **リスク最小**: 売りシグナルには影響なし

```python
df = pd.read_parquet('data/parquet/prices_max_1d.parquet')

```

## v2.0.3 最適化戦略（損切り・精算タイミング）

**リリース日**: 2025-11-20
**データ期間**: 2025-10-29 〜 2025-11-20（23 日間、146 件）
**目的**: イントラデイ分析に基づく損切り水準と精算タイミングの最適化

### バックテスト結果サマリー

**v2.0.3 基本戦略**:
- v2 スコアで買い/売り/静観判定
- 5,000-10,000 円: 強制買い
- 10,000 円以上: 強制売り

**実績**:
- 買い推奨: 77 件
- 売り推奨: 31 件（勝率 74.2%、合計利益 +368,950 円）
- 静観: 37 件

### イントラデイ分析（5 分足データ使用）

#### 買い推奨の分析

**5,000-10,000 円のロングポジション**:
- 前場精算（11:30）: より有利（+25,400 円の差）
- 理由: 高価格株が前場で上昇後、後場で反落する傾向

**その他の買い推奨**:
- 大引け精算（15:30）を推奨

#### 売り推奨の分析

**10,000 円以上のショートポジション**（7 件）:
- 損切りなし: +330,500 円
- **5%損切り: +369,500 円（+39,000 円改善）** ✅
- 6%損切り: +351,300 円（+20,800 円改善）
- 最大上昇率: 9.62%（三井金属鉱業、寄底）、寄底除く 4.87%
- 精算タイミング: 大引けが+56,500 円有利

**10,000 円未満のショートポジション**（24 件）:
- 損切りなし: +38,450 円
- **3%損切り: +36,550 円（-1,900 円、リスク管理として許容）** ✅
- 最大上昇率: 8.83%（Bitcoin Japan）
- 理由: 例外的な大幅上昇（8.83%）から保護するため、損切り必須
- 精算タイミング: 大引けが+1,500 円有利

### 最終推奨戦略

#### 【買い推奨】

1. **5,000-10,000 円**:
   - 損切り: **3%**
   - 精算: **前場引け（11:30）**
   - 理由: 前場で利確が+25,400 円有利

2. **それ以外**:
   - 損切り: **3%**
   - 精算: **大引け（15:30）**

#### 【売り推奨】

1. **10,000 円超**:
   - 損切り: **5%**
   - 精算: **大引け（15:30）**
   - 理由: 5%損切りで+39,000 円改善、大引けが+56,500 円有利

2. **それ以外（10,000 円未満）**:
   - 損切り: **3%**
   - 精算: **大引け（15:30）**
   - 理由: リスク管理（例外的な 8.83%上昇への保険）

### 損切りの考え方

**リスク管理の重要性**:
- 10,000 円未満のショートで、3%損切りは-1,900 円（約 5%）のコスト
- しかし、Bitcoin Japan（+8.83%）のような例外的ケースから保護
- 「たまたまうまくいった」に依存せず、ルールベースで運用

**損切り価格の計算**:
- 買い: `損切り価格 = 購入価格 × (1 - 損切り率)`、切り捨て
- 売り: `損切り価格 = 購入価格 × (1 + 損切り率)`、切り上げ

### 実装優先度

**Phase 1（即時実装可能）**:
1. 損切り水準の設定（価格帯別）
2. 精算タイミングの設定（前場 vs 大引け）

**Phase 2（運用開始後）**:
1. 実績モニタリング
2. 損切り発動率の確認
3. 精算タイミングの検証
