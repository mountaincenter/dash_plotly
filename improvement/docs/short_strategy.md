# ショート戦略 設計書

## ゴール

**最終目標: お金が増える**

### ステップ

```
1. ショート特化
   └─ 現状利益を確保できているショートに集中
   └─ ロングは当面禁止

2. 勝ちパターンの確立
   └─ チャートの形を理解
   └─ 得意パターンを作る

3. パターン認識の習熟
   └─ 5分足2日分 + 日足 + 出来高
   └─ 数パターンを頭に叩き込む

4. アラートシステム
   └─ 得意パターンにハマりそうな銘柄を自動検出
   └─ 朝の時点で候補を絞る

5. ファンダメンタル補強
   └─ 一致率（貸借倍率等）
   └─ 有価証券報告書 → NotebookLMでまとめ
   └─ 銘柄の背景を理解した上でトレード

6. 知識深耕 + 利益確保の両立
   └─ 学びながら稼ぐ
   └─ 稼ぎながら学ぶ
```

## 1. 対象銘柄

```

→ トレードしやすい価格帯から始める
```

### 1.2 Phase分け

| 信用売 | 制度 or いちにち | 空売り可能 |

## 2. 出来高・流動性の考え方

### 2.1 問題意識
- 出来高で絞りすぎる → 機会損失
- 絞らなすぎる → 約定しない、スプレッド大

### 2.2 提案: 2軸管理

#### 軸1: ベース流動性（リスト全体のフィルタ）

```
売買代金 = 株価 × 出来高

Tier 1: 1億円/日以上    → メイン対象
Tier 2: 1000万〜1億円   → 状況次第
Tier 3: 1000万円未満    → 急騰時のみ
```

**検討事項:**
- [ ] 過去N日の平均売買代金を使うか？
- [ ] 最低ラインをどこに設定するか？
- [ ] Tierごとにロットを変えるか？

#### 軸2: 出来高急騰（当日のエントリートリガー）

```
出来高急騰率 = 当日出来高 / 過去20日平均出来高

2倍以上  → 注目
5倍以上  → 強いシグナル（JMACS/kudan型）
```

**検討事項:**
- [ ] 前場の時点で判定するか？（9:30時点の出来高 vs 過去平均の前場出来高）

### 3.2 N日以内の最大下落

| 期間 | 平均最大下落 | 中央値 |
|------|--------------|--------|
| 1日以内 | -0.33%    | -0.68% |
| 2日以内 | -2.22%    | -1.97% |

| 5日以内 | -5.23%    | -4.22% |
| 7日以内 | -6.41%    | -5.20% |
| 10日以内 | -7.75%   | -6.31% |

### 3.3 結論

```
勝負期間: 5〜7営業日

- 勝率は日が経つほど微増（57% → 59%）
- 最大下落は5日で-5%、7日で-6%程度
- 7日以降は効果薄れる傾向
```

## 4. エントリー戦略

### 4.1 スクリーニングプロセス（バックテスト検証済み）

```
一次スクリーニング（前日）
  └─ 出来高急騰率 2倍以上（過去20日平均比）
  └─ 常習犯リストに存在

二次スクリーニング（当日・後場）
  └─ 後場に入ってから監視
  └─ 前場高値から-2%下落した時点でショートエントリー
  └─ ※後場寄付でそのまま入らない（高値更新リスク22%）
```

### 4.2 バックテスト結果

| 指標 | 値 |
|------|-----|
| 勝率 | 76.0% |
| 平均損益 | +2.94% |
| コスト後純損益 | +2.69% |
| トレード数 | 1,517回 |

**なぜ後場寄付ではダメか:**
- 後場寄付でエントリー: 勝率53.2%、平均+0.40%
- 後場で高値更新される確率: 22%（その場合-1.81%の損失）

### 4.3 エントリー条件（確定）

- [x] 常習犯リストに存在
- [x] 前日出来高急騰（過去20日平均比 2倍以上）
- [ ] 制度信用 or いちにち信用で空売り可能（要確認）
- [x] 後場に入ってから、前場高値-2%でエントリー

### 4.4 エントリータイミング

**原則: 後場に入ってから、前場高値-2%到達時にエントリー**

| 状況 | 対応 |
|------|------|
| 後場寄りで前場高値-2%以下 | 寄りでエントリー |
| 後場で前場高値-2%到達 | 到達時にエントリー |
| 後場で前場高値-2%未到達 | エントリーしない |

**やってはいけないこと:**
- 後場寄付でそのまま入る → 高値更新リスク22%
- 前場でエントリーする → 高値が確定していない

## 6. 運用フロー

### 6.1 一次スクリーニング（前日）

```
目的: 翌日の監視対象を絞る
```

1. 常習犯リスト更新（`python3 improvement/scripts/update_morning_peak_html.py`）
2. 出来高急騰銘柄の抽出（`short_candidates_tomorrow.html`）
3. 候補銘柄の確認:
   - 空売り可能か（制度/いちにち信用）
   - 材料の有無（決算、IR等）
   - 除外判断（好材料 → ショートリスク高）

### 6.2 二次スクリーニング（当日）

```
目的: 高値形成を見てエントリー判断
```

**寄り前（8:30〜9:00）**
1. 先物・米国指標の確認
2. 前日候補の気配確認

**ザラ場（9:00〜15:00）**
1. 候補銘柄の値動き監視
2. 高値形成の確認
3. **高値形成 → ショートエントリー**
4. **ロング禁止**

### 6.3 エントリー判断

| 状況 | 判断 |
|------|------|
| 候補が寄りから下落 | 高値形成なし → 見送り |
| 候補が上昇後に反転 | 高値形成 → エントリー |
| 地合い悪化（先物大幅安等） | 候補でも新規ショートは慎重に |
