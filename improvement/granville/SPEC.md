# グランビル法則 戦略検証仕様書

## 目的

日本株の短期トレードでエッジを持つ戦略を構築する。
グランビル8法則をベースに、有限資金で実行可能な戦略を導出する。

## 原則

- **モデルはシンプル、特徴量とデータ品質を上げる**
- 変数を最小にした理論値から、1つずつ変数を追加して差分を検証する
- 全ての中間結果はparquetで保存し、任意の変数を後から検証可能にする
- PnL合計は参考値。主指標は **PF(Profit Factor)** と **1トレードあたり平均損益**

## グランビル8法則 定義

### LONG (B1-B4)

| 法則 | 名称 | エントリー条件 | イグジット |
|------|------|---------------|-----------|
| B1 | ゴールデンクロス | 前日Close<SMA, 当日Close>SMA, SMA上昇 | Close < SMA |
| B2 | 押し目買い | SMA上昇, 乖離-5~0%, Close<SMA, 陽線 | Close < SMA |
| B3 | SMA支持 | SMA上昇, Close>SMA, 乖離0-3%, 乖離縮小, 陽線 | Close < SMA |
| B4 | 乖離反発 | 乖離 < -8%, 陽線 | Close >= SMA (平均回帰) |

### SHORT (S1-S4)

| 法則 | 名称 | エントリー条件 | イグジット |
|------|------|---------------|-----------|
| S1 | デッドクロス | 前日Close>SMA, 当日Close<SMA, SMA下降 | Close > SMA |
| S2 | 戻り売り | SMA下降, 乖離0~+5%, Close>SMA, 陰線 | Close > SMA |
| S3 | SMA抵抗 | SMA下降, Close<SMA, 乖離-3~0%, 乖離拡大, 陰線 | Close > SMA |
| S4 | 乖離反落 | 乖離 > +8%, 陰線 | Close <= SMA |

### 共通ルール

- エントリー: シグナル翌営業日の寄付(Open)
- イグジット: イグジットシグナル翌営業日の寄付(Open)
- 最大保有: `MAX_HOLD` 営業日で強制決済（検証対象の変数）

> **注**: グランビル理論にはExit条件（SMAクロスバック/平均回帰）のみが定義されており、
> 最大保有期間の規定はない。`MAX_HOLD` はExit条件が発火しない場合の安全弁として導入する。
> 適切な値はStep 2.5で検証する。

## 検証フロー

```
Step 1: エッジの存在確認
  全143銘柄・制約なし・SL 3パターン(なし/-3%/-5%)
  8法則すべて検証 → ショート不要の確認、レジーム効果の確認
  ↓ 死んだシグナルを除外

Step 2: ユニバース制限
  資金拘束を制御するため、銘柄ユニバースを絞る。
  以下の各ユニバースで Step 1 生存シグナルの PF を比較し、
  エッジが維持される最小ユニバースを特定する。
  各ユニバースの結果は個別に保存し、後から比較可能にする。

  検証ユニバース:
    a) Core30（31銘柄）
    b) TOPIX100 = Core30 + Large70（100銘柄）
    c) 政策銘柄（72銘柄、防衛・半導体・経済安保等。Mid400/Small含む）
    d) Core30 + 政策銘柄の和集合
    e) 全143銘柄（Step 1と同一、ベースライン）
    f) セクター除外パターン

  ↓ ユニバースを確定

Step 2.5: Exit戦略
  シグナルExit / 固定日数Exit(複数パターン) / MAX_HOLD値
  保有日数を制御して資金計画を立てる
  ↓ Exit方法を確定

Step 3: 資金制約シミュレーション
  信用余力(変数)での実行可能性
  同時保有制限、ポジションサイズ
  → 実トレード戦略を確定
```

各Stepで候補が絞られ、次のStepに持ち越すシグナルが減る。
Step Nの結果がStep N-1の前提を覆した場合、遡って再検証する。

## パラメータ一覧

検証で使用する変数。定数として固定するものと、検証対象として複数値をテストするものを区別する。

### 定数（固定）

| パラメータ | 値 | 根拠 |
|-----------|-----|------|
| 取引単位 | 100株 | 制度信用の最低単位 |
| 証拠金率 | 30% | 楽天証券 制度信用 |
| 手数料 | 0円 | 楽天証券 ゼロコース |
| エントリー | シグナル翌日Open | グランビル法則の標準適用 |
| イグジット | Exit条件翌日Open | 同上 |

### 変数（検証対象）

| パラメータ | 検証値 | 検証Step |
|-----------|--------|---------|
| SL (ストップロス) | なし / -3% / -5% | Step 1 |
| SMA期間 | 20 (基準) / 他はStep 1結果で判断 | Step 1 |
| レジーム判定 (N225 vs SMA) | SMA期間と連動 | Step 1 |
| ユニバース | 全143銘柄 / Core30 / TOPIX100 / 政策銘柄 / Core30+政策銘柄 | Step 2 |
| セクター除外 | なし / 特定セクター | Step 2 |
| Exit方法 | シグナル / 固定日数 | Step 2.5 |
| MAX_HOLD | 60 / 他 | Step 2.5 |
| 信用余力 | 実際の余力に応じて設定 | Step 3 |

## 評価指標

| 指標 | 用途 | 優先度 |
|------|------|--------|
| PF (Profit Factor) | 資金量に依存しない勝ち負け比率 | **主指標** |
| 1トレード平均損益(円) | 少数ポジでの期待値 | **主指標** |
| 勝率 | 参考(PFと合わせて判断) | 副指標 |
| 平均保有日数 | 資金回転率 | 副指標 |
| PnL合計(万) | 無限資金前提の理論上限 | 参考値 |

## 取引環境

| 項目 | 値 |
|------|-----|
| 証券会社 | 楽天証券 |

### meta.parquet ユニバース分類

`topixnewindexseries` カラム（TOPIX指数区分）と `categories` カラム（配列型）で銘柄を分類できる。

| ユニバース | 銘柄数 | データソース | 備考 |
|-----------|--------|-------------|------|
| Core30 | 31 | `topixnewindexseries == "TOPIX Core30"` | 超大型 |
| Large70 | 69 | `topixnewindexseries == "TOPIX Large70"` | 大型 |
| TOPIX100 | 100 | Core30 + Large70 | |
| 政策銘柄 | 72 | `categories` に `"政策銘柄"` を含む | 防衛・半導体・経済安保等。Core30/Large70/Mid400/Small横断 |
| Mid400 | 23 | `topixnewindexseries == "TOPIX Mid400"` | 中型 |
| Small | 7 | Small 1 + Small 2 | 小型 |
| 未分類 | 13 | `topixnewindexseries` が None | 新興・小型 |
| **全銘柄** | **143** | | |

> **注**: 政策銘柄はTOPIX区分を横断する。Core30と政策銘柄の重複は29銘柄。
> 資金拘束が重い場合に、ユニバースを絞ることで同時シグナル数を制御できる。

データ期間: 1999-07 ~ 2026-02（parquetに含まれる全期間）

## 検証期間

- **全期間**: データに含まれる全期間（構造の安定性確認）
- **直近期間**: 検証対象の変数。全期間の結果と比較して構造変化の有無を確認し、
  変化が顕著な時点をデータから特定する（恣意的な期間設定はしない）
